## 一、游戏概述：形式元素分析（Fullerton）

我首先从形式元素（formal elements）来梳理《Last Hand》的核心设计——玩家、目标、规则、资源、冲突、边界与结果。这一部分既是对最初 game proposal 的回顾.

### 1. 玩家与角色

- **玩家数量**：单人对战 AI。玩家扮演一名在边境小镇游走的赌徒，对手则是一系列风格各异的庄家 / 敌人。  
- **对手类型**：不同敌人有不同的行为倾向（贪心型、保守型、随机型），在形式元素上体现为不同的“程序性人格（procedural character）”。

### 2. 目标与胜负条件

- **短期目标（局内）**：在每一局对决中，通过抽牌接近目标点数（通常是 21，但可能受到环境规则影响），在不爆牌的前提下打败当前敌人。  
- **中期目标（Run 内）**：在一次 Run 中，连续通过多个 Level，维持自己的生命值，积累金币和道具，在惩罚规则和随机环境的压力下走得尽可能远。  
- **长期目标（Meta 进程）**：通过多次 Run 逐渐解锁生命值和物品栏的持久升级，体验不同环境、惩罚和物品组合，探索更深的策略空间。  
- **胜负条件**：  
  - 局内胜利：在一局结算时比分更接近目标点数，或对手爆牌而自己未爆。  
  - Run 结束：玩家生命归零时 Run 结束，进入 Game Over。  
  - Normal 模式的阶段性“胜利”：击败一定数量敌人后，会出现较强的敌人和更高额赏金，形成阶段高潮；Endless 模式则是纯粹的生存挑战。

### 3. 规则与流程（Procedures & Rules）

游戏的**程序（procedures)** 可以概括为：

1. **起始阶段**：  
   - 玩家从主菜单选择普通模式或无尽模式。  
   - 当前保存槽会提供本次 Run 的初始生命和物品栏上限。  
   - 每个 Level 开始时，系统为本局抽取若干“通告牌”（环境卡）和一张“惩罚牌”，并生成一名敌人。
2. **抽牌与行动阶段**：  
   - 玩家与敌人轮流行动：可以选择“要牌（Hit）”或“停牌（Stand）”，敌人根据其 AI 性格做出决定。  
   - 玩家可以在回合中使用背包里的道具牌（例如回血、改写结算伤害、重抽上一张牌、强制对方抽牌等），这些都属于改变规则执行方式的“临时程序”。  
3. **结算阶段**：  
   - 比较双方点数，判定赢家与输家。  
   - 根据当前惩罚牌的规则计算双方受到的伤害与回复（例如定额伤害、按分差伤害、连胜涨伤害、胜者小幅回血等）。  
   - 结算完毕后，根据环境牌的特殊规则额外应用效果（如突然死亡阈值、完美奖励、爆点阈值变化）。  
4. **奖励与进程阶段**：  
   - 击败敌人后玩家获得金币与战利品选择界面，可以从数张道具牌中选择若干加入背包。  
   - 随着 Level 上升，敌人的生命和防御提高，环境与惩罚组合也更极端。  
   - 当 Run 结束时，系统将本次赢得的金币结算到 Meta 经济中，用于在主菜单购买持久升级。

### 4. 资源与风险结构（Resources）

在资源设计上，我们试图构造一个围绕 “风险与补偿” 的系统：

- **生命值（HP）**：最直接的生存资源。惩罚牌和伤害规则让每一局的输赢都真正“肉疼”。  
- **手牌与牌堆**：传统 Blackjack 的基础资源，控制你当下能否逼近目标点数。  
- **道具牌（Items）**：一次性资源，可以改变抽牌、结算和伤害逻辑，是构建策略差异的关键。  
- **环境规则（Environment Cards）**：改变全局规则的“公共资源”，例如固定 A 为 11 点、自动抽牌、目标点数改为 18/24/27、禁止使用道具等。  
- **惩罚牌（Penalty Cards）**：Run 级别的“威胁资源”，它们定义“输一局要付出什么代价”，形成持续的心理压力。  
- **金币与 Meta 升级**：跨 Run 的经济资源，提供生命上限和道具栏位的成长，让长线目标与短期决策产生联系。

### 5. 冲突、边界与结果

- **冲突来源**：  
  - 与敌人点数竞赛的直接对抗。  
  - 在“继续要牌”与“停牌等待对方犯错”之间权衡。  
  - 在当前 Run 的生命安全与未来 Run 的经济成长（金币、道具）之间权衡。  
- **边界**：  
  - 场景上，所有行动都发生在赌桌与小镇的虚构空间中。  
  - 规则上，玩家行动被严格限制在抽牌、停牌、用道具三个基本操作，加上少数由环境触发的自动效果。  
- **结果**：  
  - 局内清晰的输赢反馈（胜利动画、爆牌提示、伤害数字）。  
  - Run 结束后的“BUSTED”画面和金币结算，为下一次尝试埋下钩子。  

整体来看，《Last Hand》的形式元素围绕一个清晰的核心问题展开：**“你愿意为多一张牌、一个更贪心的决策付出多大代价？”**

## 二、戏剧元素与体验目标（Dramatic Elements）

单有规则还不足以构成完整体验，游戏需要戏剧元素（dramatic elements）来承载情绪与叙事。我们的目标是把一个抽象的概率博弈包裹在“最后一局牌”的故事里。

### 1. 主题与前提（Premise）

游戏的核心前提是：**在一座边境小镇里，玩家用最后一手牌与各路危险人物对赌命运。**  
这一前提支撑了以下几点设计：

- 赌桌视角与像素化小镇背景，强化“封闭而危险”的空间感。  
- 惩罚牌像是挂在酒吧墙上的规则告示，让玩家在每次翻开它时都能感到一丝不安。

### 2. 挑战曲线与节奏（Challenge & Dramatic Arc）

- **单局内部**：  
  - 开局双方各两张牌，节奏比较平稳。  
  - 中后期在关键几张牌和道具使用点上形成小高潮：一次精准抽牌或一次爆牌，都会明显改变局势。  
  - 结算阶段通过伤害数字和文字提示制造“落差瞬间”，让胜负体感不只是数字变化。  
- **Run 级别**：  
  - 随着 Level 升高，惩罚规则和环境效果逐渐叠加，形成难度爬坡。  
  - 中间穿插的奖励阶段（战利品选择）和主菜单的升级界面，则作为“呼吸点”，让玩家在压力与恢复之间循环。  
- **无尽模式**：  
  - Endless 模式去掉了明确的终点，更像是 Fullerton 所说的“耐力挑战”（endurance challenge），考验玩家如何在长期资源管理与即时决策之间保持冷静。

### 3. 情绪目标

在戏剧层面，我们希望玩家经历的情绪轨迹包括：

- 每局开局时的**好奇与谨慎**：先观察环境与惩罚规则，再决定自己要多激进。  
- 回合中不断抽牌的**贪心与焦虑交替**：特别是在接近爆点时，玩家会反复权衡“再来一张”的诱惑。  
- 惩罚牌结算时的**懊悔或报复快感**：例如因为连胜而让对手吃到更高伤害，或自己被“按分差伤害规则”惩罚。  
- Run 失败时的**挫败混合释然**，以及回到主菜单看到金币累计时的“下一把会更强”的期待。

## 三、设计与开发迭代

作为队伍中唯一的程序实现者，下面按阶段梳理关键决策与变化。

### 1. 初始概念阶段

在最早的 game proposal 中，我们提出的核心愿景是：

- 把 Blackjack 的紧张感扩展成一套 Rogue-lite 式的长程体验，而不是一局一结。  
- 用“环境牌 + 惩罚牌 + 道具”三层结构，制造 Run 之间和局与局之间的差异性。  
- 通过 Meta 升级提供缓慢但稳定的进度，让失败也“有产出”。  

### 2. 规则与系统搭建阶段

在开始实现之前，我们先把系统拆成几块相对独立的设计问题：
- **抽牌与计分规则**：  
  - 采用接近 Blackjack 的基础规则（A 可变点数、爆牌线、目标点数等），同时为后续环境规则留出空间（例如 A 固定为 11、改变爆点、随机目标分数等）。  
- **惩罚系统**：  
  - 设计了若干种不同的伤害曲线：定值伤害、按分差伤害、随回合数递增、连胜叠加等，让“输一局”的代价既可预测又会变化。  
- **环境系统**：  
  - 环境牌重点改变“怎样才算好手牌”，例如完美得分追加奖励、自动抽牌、禁止道具、特殊爆点等。  
- **道具系统**：  
  - 设计一批一次性道具，分别干预抽牌、结算和资源（治疗、护盾、改写目标分数、从牌库找特定点数等），目的是让玩家在每回合有比“要 / 不要牌”更丰富的选择。  
- **Meta 经济**：  
  - 通过金币购买生命值上限和物品栏位，让失败的 Run 仍然推动长线成长，缓和难度曲线。

这一阶段的主要挑战，是在纸面上平衡这些系统的组合；作为程序实现者，我需要不断提醒自己：**系统之间的组合复杂度会在试玩时被放大**。

### 3. 试玩驱动的迭代阶段

在实现出可玩的内测版本后，我们经历了若干轮以试玩为核心的迭代，大致可以分为三个重点：

1. **清晰度与可读性迭代**：  
   - 早期版本里，玩家很难区分“环境规则”和“惩罚规则”，常常不知道自己为什么突然多掉了几滴血。  
   - 根据反馈，我们增加了更醒目的文案提示、图标和分区布局，让玩家能在一两局内理解三大系统的分工。  
2. **难度与节奏迭代**：  
   - 在一开始的设定中，伤害数值略高、回血手段太少，导致 Run 经常在 2～3 局内就黯然结束。  
   - 后续通过调整定值伤害、提高部分治疗道具的出现频率，以及在完美得分时给予额外奖励，才逐步把体验拉到“紧张但不是绝望”的水平。  
3. **策略空间与“骗术”感迭代**：  
   - 试玩者反馈道具很有趣，但有些局面下感觉“用不用都差不多”，缺乏明显的策略反差。  
   - 我们因此新增 / 调整了几种更极端的道具与环境效果（例如强制敌人抽牌、改变胜负判定分数、限制道具使用），让玩家在关键几回合能做出真正有风险的“大动作”。

## 五、代表性试玩观察

### 1. 入门与规则可读性

**观察：**

- 新玩家在第一局经常被三层系统（环境、惩罚、道具）同时砸到头上，会有明显的信息过载。  
- 不少人知道“抽到接近目标分数就好”，但不清楚完美得分的额外奖励，也不总能注意到惩罚牌的描述。  
- 一部分玩家在前两局完全忽略道具牌，把它们当作单纯的“花哨装饰”，直到看到严重受伤才开始尝试使用。

**分析：**

- 我们在最初版本中把太多 formal elements 同时抛给了玩家，缺乏循序渐进的“教程层”。  
- 信息优先级不够明确——哪些是“这一局必须知道的规则”（目标分数、惩罚牌），哪些是“可以慢慢摸索的深度策略”（特定道具组合）。  

**已有或计划中的调整：**

- 增强教程的分步呈现，先解释基本抽牌与爆牌规则，再逐步引入环境和惩罚。  
- 在前几局中限制环境与惩罚的组合数，让玩家先熟悉一两种典型模式。  
- 提升第一次使用道具时的反馈强度，让玩家意识到“这是一张能改写命运的牌”。

### 2. 决策空间与策略多样性

**观察：**

- 有经验的卡牌玩家很快就形成了固定套路：  
  - 若当前惩罚是“固定伤害”，他们会倾向于稳定保守，避免大起大落；  
  - 若惩罚与分差挂钩，则更愿意“要到极限”，争取一次性重创敌人。  
- 一些环境组合（例如自动抽牌 + 固定高点数 A）会让难度陡增，玩家反馈：“仿佛这局从一开始就是输家”。  
- 部分道具（如“抽取最优牌”“精确点数抽牌”）被认为过于强力，玩家会倾向于“攒着等关键时刻用”，结果很多 Run 结束后仍有一堆未用的强卡。

**分析：**

- 总体上，游戏确实提供了较丰富的决策空间，但有几个问题：  
  - 某些环境组合使得最优策略高度单一（例如只能极度保守或极度激进），降低了选择的乐趣。  
  - 强力道具的“机会成本”感太模糊，玩家看不到不用它会失去什么。  

**调整方向：**

- 为部分极端环境增加“对称补偿”，例如自动抽牌环境下提升治疗或护盾资源的出现率。  
- 在结算与日志中更明显地标注“如果这里用了某某类型道具，结果会如何不同”，帮助玩家事后反思策略。  
- 调整强力道具的数量与获取方式，让它们更像是“稀有而决定性的赌注”，而不是“永远留着的保险箱”。

### 3. 难度曲线与公平感

**观察：**

- 初次试玩的玩家普遍觉得前两局偏难，常常在还没完全理解系统时就被高额惩罚击倒。  
- 有人反馈某些局“感觉系统在针对我”，特别是在连续几轮自动抽到坏牌或环境组合极端时。  
- 有经验的玩家则在适应几局后，能稳定通过前几个 Level，并认为中后期的难度更有趣。

**分析：**

- 一方面，这是 Rogue-lite 类游戏常见的“初见暴死”；另一方面，我们的伤害曲线与治疗手段在早期版本中确实偏于严苛。  
- 当玩家缺乏对牌堆状态和概率的可见性时，很容易把短期波动解读为“系统不公平”，而不是随机性的自然结果。

**调整方向：**

- 下调早期惩罚牌的伤害上限，并提升前几局中治疗与防御道具的比重。  
- 提供更多“中间反馈”，例如在日志里提示“本局已经出现了多少高点数牌”，帮助玩家形成直觉。  
- 在 Meta 升级上给予更明显的“保底感”，例如早期可以更快解锁 +HP 或额外道具栏。

### 4. 情绪节奏与反馈系统

**观察：**

- 多数试玩者对“爆牌”“完美得分”“突然死亡”等瞬间情绪反馈强烈，认为是本作最有记忆点的时刻。  
- 部分玩家会在连胜后明显“飘起来”，开始做更激进的选择；对应的连胜加伤惩罚牌能有效把这种情绪转化为紧张感。  
- 也有人提到，当连续几局都不出现“戏剧性事件”时，中间段略显平淡。

**分析与方向：**

- 这些反馈印证了 Fullerton 对“戏剧弧线”的重视：我们需要在长线 Run 中穿插更多的小高潮。  
- 后续可以考虑：  
  - 为特定条件下的逆转或险胜设计额外演出与奖励。  
  - 调整事件触发概率，让“有戏剧性的局面”在一段时间内必然出现一次。

### 5. 系统边缘情况与可预期性

**观察：**

- 复杂道具叠加环境与惩罚时，玩家有时很难预判最终结果，例如：  
  - 先改变目标分数，再用护盾、再触发连胜加伤，这几层效果如何叠加。  
  - 在同时存在“分差伤害”和“突然死亡阈值”时，先应用哪一个规则。  
- 虽然系统内部有清晰的结算顺序，但从外部体验上，这部分知识需要较长时间才能自发习得。

**调整方向：**

- 在关键结算时刻提供更详细的“逐条说明”，例如用一段自然语言简要解释这回合伤害的来源。  
- 在教程或帮助界面加入“结算顺序示意图”，降低学习成本。

## 六、困难、成功与个人反思

### 1. 主要困难

- **系统组合爆炸**：  
  - 环境、惩罚和道具三层系统一旦组合，理论上的局面数远超我们能一一验证的范围。作为程序实现者，我需要在逻辑上保证“任何组合都不会把游戏玩坏”。  
- **早期玩家体验的脆弱性**：  
  - 当新玩家在前两局遭遇极端组合时，很容易直接放弃。如何在保留系统深度的同时，给新手留一道安全缓冲，是我们一直在权衡的问题。  
- **时间与范围的权衡**：  
  - 我们在 proposal 中设想了更完整的 Level 地图系统、商店节点、宝箱节点等内容，但在现实开发中不得不收缩范围，把精力集中在战斗核心上。

### 2. 阶段性成功

- **核心循环扎实**：  
  - 大部分试玩者在理解规则后，会主动想要再来几局，说明“抽牌 → 结算 → 奖励 → 升级”的核心循环是有黏性的。  
- **主题与机制统一**：  
  - 许多反馈会直接用“贪心”“赌一把”这样的词来形容自己的行为，这说明主题确实通过机制传达到玩家心里。  
- **系统可扩展性良好**：  
  - 从设计角度看，环境、惩罚和道具都可以在不改动底层规则的前提下扩充，为后续版本留下了余地。

## 七、被搁置的点子与取舍

在从 proposal 走向可玩版本的过程中，我们不得不把不少有趣的想法留在“会议室地板上”。这里列出几个对我个人影响最大的取舍：

1. **关卡地图（Level System）**  
   - 最初我们设计了一套类似《杀戮尖塔》的地图：每次通关后在分支节点之间选择下一站（普通战斗、商店、宝箱等），并只展示下一步而非整张地图。  
   - 这一系统可以在 Run 级别加入更多“路线规划”的策略，但实现成本较高，最终我们只保留了线性 Level 进程与随机环境 / 惩罚组合。  
2. **商店与宝箱事件**  
   - 设计稿中曾经包括“商店节点”和“宝藏节点”，前者允许用金币购买特定道具，后者则在奖励与陷阱之间摇摆。  
   - 由于时间限制，这些事件目前只以简单的奖励与 Meta 升级形式出现，缺少独立的地图层结构。  
3. **更细致的敌人个性化**  
   - 我们原计划为每个敌人配套独特的环境与惩罚组合，让他们在玩法上更鲜明，例如“只偏爱防御型道具的法官”“会偷用你道具的幽灵”等。  
   - 目前的版本中，敌人个性主要体现为抽牌倾向和基础属性，真正的“人格差异”还有待拓展。  
4. **更强的叙事包装**  
   - 在故事层面，我们一度考虑加入小镇 NPC、分支对话和长期因果（例如在某家酒馆赢太多会触发特别 Boss）。  
   - 最终这些内容只能以 UI 和文案的碎片形式存在，没有上升为完整的叙事系统。

这些被搁置的想法一方面提醒我团队范围控制的重要性，另一方面也为后续版本提供了清晰的演进路线。

## 八、基于本次报告的修改计划

1. **重新设计新手体验层级**  
   - 在前几局中限制环境与惩罚的复杂度，以“只出现一种环境 + 一种惩罚”为主，逐步解锁更多组合。  
   - 在教程中用更明确的步骤呈现：先讲抽牌与爆牌，再讲惩罚，再讲道具与环境。  
2. **显性化关键结算逻辑**  
   - 在结算界面加入简短的“文字分解”，例如：“你输了本局，惩罚为按分差伤害，分差 4 点 → 受到 4 点伤害”。  
   - 在帮助界面绘制“结算顺序”简图，降低玩家对系统黑箱的感受。  
3. **调整早期难度与资源节奏**  
   - 适当降低 Level 1–2 的惩罚强度，并提升治疗 / 防御道具的出现率，让新手有时间犯错。  
   - 增强完美得分在早期的奖励效果，引导玩家积极尝试更精细的决策。  
4. **丰富长程策略与敌人差异**  
   - 在不改动核心规则的前提下，为部分敌人绑定特定的环境或惩罚组合，让他们在体验上形成“套路”。  
   - 后续如果时间允许，逐步把地图、商店和宝箱系统重新引入，使 Run 的规划不再只是“下一局打不打得过”，而是“我要走哪条风险与收益不同的路”。  
5. **利用 Meta 进程平滑学习曲线**  
   - 调整升级价格与收益，让新玩家在几次失败后能够明显感受到“下一次会更强”，而不是停留在挫败感中。
